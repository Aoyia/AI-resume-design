# 个人简历制作网站 — 产品需求文档（PRD）

> **项目目标**：打造一款交互极其简洁、产出效果对标最高级企业标准（如超级简历）的个人简历制作网站。  
> **核心理念**：无冗余流程，打开即用，直接输出最高质量的 PDF 简历与超高清图片。

---

## 一、产品定位与核心原则

| 维度 | 描述 |
|------|------|
| **目标用户** | 求职者（应届、社招） |
| **核心价值** | 最快速度产出一份专业、排版完美的简历 |
| **交互原则** | **无需首页，直接进入编辑器页面**；无需强制登录即可制作 |
| **产出标准** | **最高质量的矢量 PDF**（文字可选中，ATS 友好） 与 **Retina 高清 PNG 图片** |
| **平台支持** | **仅考虑 PC 端**（桌面级浏览器），专注复杂编辑的体验 |

### 2.2 编辑面板与表单设计
* **极简大纲（去卡片化）**：消解了原本多层灰色卡片的视觉拥挤，左侧面板整体纯白，模块以极其淡雅的单分割线区分。
* **手风琴式折叠**：所有大模块（如教育经历、工作经历、项目经历等）均支持收起与展开，并且支持通过拖拽手柄进行全局顺序调整。
* **悬浮唤醒（Hover Reveal）**：拖拽手柄 `⋮⋮` 和删除等管理图标默认隐藏，仅当鼠标移入该大模块/经历项时才平滑出现，平时输入视线没有任何噪点。
* **无边框输入框（去格子化）**：文本框与输入框移除了默认四周框线，仅表现为极淡融合背景；在用户聚焦（Focus）输入时才浮现精致的主色外轮廓，保持极强的撰写专注力。
* **输入防抖优化**：输入采用局部状态缓存 + 150ms 防抖同步，彻底解决复杂表单打字延迟，保持极致输入流畅度。

### 2.3 实时预览（A4 真实多页拼接视图）
* **真实多页物理渲染**：废除了单张长纸辅助线的做法。引入了 **DOM 动态测量高度算法**，将简历内容按照 A4 可容纳的最大内容高度进行实时精确分页拆分，在网页上直接渲染出**多张严格 `794px * 1123px` 独立大小、具有质感阴影和页码指示器、上下紧凑排列的 A4 纸张页面**。
* **单项经历防截断**：通过在分页计算中识别经历项块，并使用 CSS `@media print` 中的 `break-inside: avoid` 原则，避免任何一段经历或段落被腰斩。

## 三、双导出方案（高保真 PDF & 超高清 PNG）

为达到“对标超级简历”的严苛效果，本项目放弃了任何 html2canvas / html2pdf.js 等有损的前端截图转图片方案。

### 3.1 服务端 Puppeteer 矢量 PDF 生成
* **原理**：在 Node.js 服务端启动真实的无头 Chrome 浏览器，跳转到专用的 A4 分页打印路由 `/print?id=xxx`。
* **优势**：打印渲染引擎原生输出全矢量 PDF，文字可选择复制，放大 500% 无任何缩放锯齿，对 CSS `@media print` 样式有极其精准的控制。

### 3.2 高清图片（PNG）导出
* **原理**：同样利用服务端 Puppeteer 无头浏览器。通过配置 **`deviceScaleFactor: 4` (4倍超清物理分辨率，印刷级超级 Retina 级别)** 的物理视口进行整体元素（`.print-wrapper`）捕获，最终渲染出极致清晰的高保真 PNG 图片并返回。
* **优势**：由于分辨率翻倍，截出的图片极其锐利精细，文字放大数倍依然清晰可辨，彻底解决了用户在手机上分享时由于社交平台二次压缩导致简历模糊的实际痛点，效果顶级。

### 3.3 精准一致性闭环
* 打印页面采用与预览完全一致的 DOM 动态高度测量进行强制分页。重组分页完毕后，标注 `data-ready="true"`。
* 服务端 Puppeteer 导出时会执行 `await page.waitForSelector('[data-ready="true"]')`，确保在无头浏览器中百分之百等到页面重组分页就绪后才启动 PDF 转换或截图，实现了前端预览与服务端导出 **100% 绝对一致（所见即所得）**。

## 四、用户系统与云端同步

| 功能 | 描述 |
|------|------|
| **低门槛使用** | **核心规则**：完全可以不登录就制作简历并下载。 |
| **本地先行容灾** | Zustand 使用 `persist` 中间件，默认将用户的修改全部暂存在浏览器 `localStorage`（键名 `resume-storage`），防丢容灾。 |
| **极简登录** | **暂不使用邮箱注册或 OAuth**。提供统一的「简单登录弹窗」（输入预设的简单账号密码即可，如：`admin` / `123456`，用于演示和个人后续同步）。 |
| **云端保存** | 只要完成登录，内容修改即触发自动保存，并模拟向轻量数据库进行备份。 |

## 五、技术方案概要

### 5.1 技术栈选型

| 层级 | 技术 |
|------|------|
| **框架** | Next.js (React) + TypeScript |
| **样式** | Vanilla CSS / CSS Modules（自定义 CSS 变量与打印控制） |
| **渲染引擎** | DOM 动态高度测量分页算法 + Puppeteer 后端导出 |
| **状态管理** | Zustand（维护全局简历的 JSON 数据，细粒度 Selector 订阅） |
| **拖拽库** | `@dnd-kit/core` 与 `@dnd-kit/sortable` |

### 5.2 高扩展性架构设计

尽管目前只有**一套简约默认模板**，但底层核心数据 `ResumeData` 必须与 `TemplateRenderer` 组件完全解耦。新增模板时，只需新增一个 `TemplateB.tsx` 组件去读取同一套 `ResumeData` 即可，实现 OCP（开闭原则）。
