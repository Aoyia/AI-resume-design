# 前端详细设计

## 目标

本目录用于定义前端实现方案，目标不是产品介绍，而是让开发人员可以直接按文档拆文件、写组件、接状态、接接口并完成联调。

前端实现必须满足以下约束：

- 仅支持 PC 端，最小设计宽度按 `1440px` 处理。
- 用户访问 `/` 直接进入简历编辑器，无首页。
- 未登录也能编辑和导出，本地草稿默认持久化。
- 登录后自动开启云端同步。
- 右侧预览必须保持 A4 视觉比例，并支持多页预览。
- PDF 导出链路以前端触发、后端生成、浏览器下载为准。

## 文档清单

- `01-application-architecture.md`：前端总体架构、路由、模块边界、建议文件树。
- `02-domain-model-and-state.md`：共享数据模型、Zustand store 设计、持久化和同步策略。
- `03-editor-interaction-design.md`：编辑器页面、表单模块、拖拽、交互状态、组件拆分。
- `04-preview-and-template-system.md`：预览区、模板引擎、分页策略、打印共享渲染约束。
- `05-client-api-and-pdf-flow.md`：前端 API 调用层、登录/同步/PDF 下载完整链路。
- `06-frontend-test-checklist.md`：单元、集成、E2E 验收清单。

## 建议实施顺序

1. 先建立 `src/shared` 下的类型和 schema，避免表单、预览、接口各自定义结构。
2. 完成 `src/store` 与编辑器页面骨架，确保左侧编辑和右侧预览用同一份状态。
3. 先做基本信息、教育经历、工作经历三个模块，打通数据更新和预览渲染。
4. 再接入 `dnd-kit`，完成模块排序和条目排序。
5. 完成模板渲染和分页预览后，再接登录、同步、导出。
6. 最后补齐错误处理、空状态、测试和视觉打磨。

## 交付标准

- 开发人员无需再反向猜测字段含义、状态边界、接口行为。
- 每个大模块都能找到对应文件拆分建议。
- 前端和后端对同一份 `ResumeData` 结构理解一致。
